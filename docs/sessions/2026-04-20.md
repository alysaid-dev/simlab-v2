## Root cause 403 Laboran/Dosen/Kepala Lab

`backend/src/middleware/auth.ts:350` — `requireRole` hanya memanggil
`deriveRoles(req.user)` yang sumbernya `memberOf` + `affiliation` + env
`superAdminUids`. Tabel DB `user_roles` (diisi via `PUT /api/users/:id/roles`)
**tidak dibaca**. Alhasil user yang di-assign LABORAN/DOSEN/KEPALA_LAB via UI
Manajemen Pengguna → role-nya tidak efektif → 403 saat approve/reject.

### Fix

- `ShibbolethUser` +`dbRoles?: Role[]` — di-populate sekali saat
  `resolveShibbolethUser` (bukan per-request), di-cache di `sessionCache`
  (TTL 60s).
- `deriveRoles()` gabung:
  1. `memberOf` + `affiliation` via `ROLE_MAP`.
  2. `user.dbRoles` (DB).
  3. Auto `MAHASISWA` kalau email endsWith `@students.uii.ac.id`.
  4. `SUPER_ADMIN` kalau uid ada di `env.shibboleth.superAdminUids`.
- `fetchDbRoles(uid)` helper di `auth.ts` — JOIN users → user_roles → roles.

## MAHASISWA role

Enum `RoleName` +`MAHASISWA` via `prisma db push`. Row seed di tabel `roles`.

`upsertFromShibboleth` (users.service) tambahan: kalau email
`@students.uii.ac.id`, auto-upsert `user_roles` dengan role MAHASISWA. Jadi
mahasiswa yang login pertama kali langsung punya role di DB (muncul di modul
Akun dengan label "Mahasiswa").

Backfill 3 mahasiswa existing dilakukan via script one-off.

## Penghapusan role ADMIN

- DB: `DELETE user_roles WHERE role=ADMIN` (2 rows), `DELETE role ADMIN`.
- Schema: enum `RoleName` drop `ADMIN` + `prisma db push --accept-data-loss`.
- Code:
  - `auth.ts`: `Role` type, `ROLE_RANK` (SUPER_ADMIN turun ke 5), `ROLE_MAP`
    (buang keys `admin`, `simlab-admin`).
  - Semua route guard yang sebelumnya memuat `"ADMIN"` → dihapus entry-nya.
  - `users.routes.ts`: `requireRoleAtLeast("ADMIN")` → `"LABORAN"` (modul
    Akun jadi wewenang Laboran).
  - `equipment.routes.ts` delete → `"SUPER_ADMIN"` (destructive).
  - `consumables.routes.ts` delete → `("SUPER_ADMIN")`.
- Frontend: `AuthContext`, `moduleAccess`, `Akun.tsx` semua dibersihkan. Akun
  dapat badge warna untuk MAHASISWA (hijau), drop badge Admin.

## Module access per role

| Role | Modul |
|---|---|
| MAHASISWA | Peminjaman Laptop, Peminjaman Ruangan, Surat Bebas Lab |
| DOSEN | Peminjaman Ruangan, Persetujuan Dosen |
| LABORAN | Peminjaman Ruangan, Persetujuan Laboran, Transaksi, Peminjaman Alat, Aset, Habis Pakai, Transaksi Habis Pakai, **Akun** |

`ProtectedRoute` pakai `canAccess(location.pathname, user.roles)` — ketik URL
langsung yang tidak diizinkan → redirect ke `/dashboard`. Dashboard tile juga
filter via `canAccess`.

Multi-role otomatis: user dengan DOSEN + KEPALA_LAB dapat gabungan modul.

## Onboarding nomor WA

- Backend: `PATCH /api/users/me` — self-update (body: `{ waNumber }`). Tidak
  butuh role minimum, hanya `requireAuth`.
- Frontend: `src/components/WaNumberOnboardModal.tsx` — modal blocking
  (non-dismissible) di App root. Muncul saat `user && !dbUser.waNumber`.
  Submit PATCH `/users/me` → refetch → modal hilang.

  mengirim waNumber di body. Sekarang first-time users forced isi dulu.
