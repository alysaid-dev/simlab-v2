## 1. Surat Permohonan Peminjaman Ruangan jadi opsional

User minta ubah field "Unggah Surat Permohonan *" di tab Pengajuan Baru
modul Peminjaman Ruang — jadi tidak wajib, tanpa label "(opsional)".

Investigasi awal: backend `reservations.controller.ts:152-156` punya
guard `if (!file) throw new HttpError(400, ...)` sehingga FE-only relax
tidak cukup; request tanpa file akan tetap ditolak server. Multer
`fileFilter` tetap memvalidasi tipe & ukuran kalau ada file, jadi cukup
hapus guard wajib-nya saja.

### Frontend — `src/pages/PeminjamanRuangan.tsx`

- Hilangkan `<span className="text-red-500">*</span>` dari label
- Hapus `required` di `<input type="file">`
- Hapus block `if (!suratFile) { setSuratError(...); return; }` di `handleSubmit`
- Kondisikan `data.append("surat", suratFile)` jadi `if (suratFile) data.append("surat", suratFile)`
- Hapus `!suratFile` dari `disabled` button submit

### Backend — `backend/src/controllers/reservations.controller.ts`

```ts
// Surat permohonan opsional — multer fileFilter tetap memvalidasi tipe & ukuran kalau ada.
const file = (req as typeof req & { file?: Express.Multer.File }).file;

const { waNumber: _waNumber, ...rest } = body;
const reservation = await reservationsService.create({
  ...rest,
  userId: requester.id,
  suratPermohonanPath: file
    ? path.relative(path.resolve("."), file.path)
    : undefined,
});
```

Service-nya (`reservations.service.ts:90`) sudah support
`suratPermohonanPath?: string` jadi tidak perlu sentuh.

### Verifikasi UI reviewer

`PersetujuanLaboran.tsx:492` & `PersetujuanKepalaLab.tsx:581` sudah
render dengan guard `r.suratPermohonanPath ? ... : ...`. Tidak perlu
patch — kalau null cukup tampil "-", kalau ada baru tombol Lihat Surat
muncul.

### Deploy

Build FE + restart BE. User jalankan `sudo systemctl restart simlab-v2-be`
sendiri (sudo butuh password tty). Status `active`.

## 2. Tab "Peminjaman Alat" di modul Riwayat

User minta tambah tab Peminjaman Alat di Riwayat Saya (super admin) +
Riwayat Lab (Kepala Lab). Modul `History.tsx` memang sudah generic untuk
dua role — judul flip antara "Riwayat Saya" vs "Riwayat Lab" (line
148-150) dan `KEPALA_LAB` di-treat sebagai `isAdmin: true` di
`history.controller.ts:28` (bypass scoping → lihat semua). Jadi tambah
1 tab langsung ke-cover dua role.

### Diskusi 3 keputusan desain

| Pertanyaan | Default saya | Keputusan user |
|---|---|---|
| Render multi-item | 1 baris/loan + badge `+N lainnya` | ✅ setuju |
| Filter Kepala Lab kalau loan campur multi-lab | tidak dedup | ✅ tidak dedup |
| Timeline modal | skip — pakai pola "Detail" Barang Keluar | ✅ opsi (a) skip timeline |

User sempat tanya: kalau skip timeline, apa detail bisa tetap dilihat
seperti Barang Keluar? Verifikasi `History.tsx:426-530` (modal
`selectedBucket`) — modal Barang Keluar tidak pakai timeline, dia render
header info + tabel item langsung. Persis cocok untuk equipment loan
yang multi-item.

Pertimbangan lain yang ikut terjawab:
- `EquipmentLoan` tidak punya `equipment_loan_status_history`, jadi
  timeline-style modal akan butuh skema audit baru. Skip jauh lebih murah.
- `EquipmentLoan` tidak punya `laboratoryId` langsung; lab di-derive
  via `items.equipment.laboratoryId`. Karena Kepala Lab di-treat
  `isAdmin: true` (lihat semua), tidak perlu filter per-lab → tidak
  ada masalah multi-lab dedup.
- `EquipmentLoan` tidak punya field `laboranHandoverBy` / `laboranReturnBy`,
  jadi non-admin scope = peminjam sendiri saja (lebih sederhana dari
  pola `Loan` laptop).

### Backend (3 file)

`services/history.service.ts`:
- Import `EquipmentLoanStatus`
- `EQUIPMENT_LOAN_FINAL_STATUSES = [RETURNED, REJECTED, CANCELLED]`
- `EQUIPMENT_LOAN_HISTORY_INCLUDE` — `user` + `items.equipment` (id, name, code, category)
- Method `listEquipmentLoans` baru — non-admin filter `userId: scope.userId`

`controllers/history.controller.ts`:
- Handler `listEquipmentLoans` (boilerplate sama dengan listLoansTA dst.)

`routes/history.routes.ts`:
- `router.get("/loans/equipment", historyController.listEquipmentLoans)`
- Diletakkan di antara `/loans/practicum` dan `/loans/:id/timeline`

### Frontend (`src/pages/History.tsx`)

- `ViewKey` tambah `"equipment"`
- `BASE_SIDEBAR_ITEMS` tambah `"Peminjaman Alat"` (antara Laptop Praktikum dan Surat Bebas Lab)
- `VIEW_BY_LABEL["Peminjaman Alat"] = "equipment"`
- `TimelineView` exclude `"consumables" | "equipment"` (skip timeline)
- Konstanta `EQUIPMENT_ENDPOINT = "/api/history/loans/equipment"`
- Types baru: `EquipmentLoanItem`, `EquipmentLoanStatus`, `EquipmentLoanRecord`
- State baru: `equipmentLoans`, `selectedEquipmentLoan`
- `loadList` tambah branch `view === "equipment"` (3-way: consumables / equipment / default)
- `openDetail` early-return juga untuk `view === "equipment"`
- `tableCaption` count include equipment
- Render branch baru di antara consumables dan default — tabel kolom: NIM, Nama Peminjam, Alat (badge `+N lainnya`), Total Qty, Status, Tgl Pinjam, Tgl Kembali, Detail
- Modal detail baru `selectedEquipmentLoan` — pola persis Barang Keluar tapi field-nya: peminjam, NIM, status badge, diajukan, tgl pinjam, tgl kembali, catatan, + tabel item (nama, kode, kategori, jumlah)

### Deploy
