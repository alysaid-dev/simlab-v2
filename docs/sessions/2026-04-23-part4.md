# 2026-04-23 part 4 ‚Äî Modul Petunjuk per role + Manajemen Petunjuk (CMS markdown)

## Konteks & keputusan

User minta modul "petunjuk penggunaan" untuk 3 audience: mahasiswa,
dosen, tendik. Plus minta super admin bisa "posting seperti website"
tanpa harus push commit. Diskusi awal pertimbangkan opsi A (markdown
static di repo) vs B (CMS DB-backed). User pilih B karena sistem
masih aktif berkembang dan tendik/laboran yang paling tahu workflow
operasional ‚Üí harus bisa update sendiri.

Keputusan akhir setelah dialog:

| Tanya | Jawab |
|---|---|
| 1 modul atau 3? | **3 terpisah** (sidebar lebih jelas, RBAC clean) |
| Per audience: 1 dok atau banyak tab? | Banyak tab/section per audience |
| Butuh upload gambar? | Ya |
| Versioning? | **Light** ‚Äî append-only snapshot tanpa UI; cukup buat safety net rollback via DB |
| Akses reader? | **Strict per role** (bukan terbuka untuk semua user login) |
| "Tendik" maps ke role mana? | **STAFF** (sudah ada di RoleName enum, tidak butuh role baru) |

Mapping section yang user sebutkan:
- **MAHASISWA** (4): Peminjaman Laptop TA, Peminjaman Laptop Praktikum,
  Peminjaman Ruangan, Penerbitan Surat Bebas Lab
- **DOSEN** (2): Persetujuan Laptop, Peminjaman Ruangan
- **STAFF** (1): Pengajuan Peminjaman Ruangan

## Implementasi

### Schema (3 tabel baru)

`backend/prisma/schema.prisma`:

```prisma
enum GuideAudience { MAHASISWA  DOSEN  STAFF }

model Guide {
  id, audience, slug, title, contentMd, order, isPublished,
  updatedById, createdAt, updatedAt
  @@unique([audience, slug])
}

model GuideSectionRevision {
  id, guideId, contentMd, updatedById, createdAt
  // append-only snapshot setiap kali content berubah
}

model GuideImage {
  id, filename (unique), path, mimeType, sizeBytes,
  uploadedById, uploadedAt
}
```

User table tambah back-relations: `guidesUpdated`,
`guideRevisionsAuthored`, `guideImagesUploaded`.

Migration via `npx prisma db push` (bukan migrate dev ‚Äî cukup untuk
dev iteration; migrate proper bisa dibuat nanti kalau perlu rollback
record).

### Seed (`backend/prisma/seedGuides.ts`)

Idempotent, upsert by `(audience, slug)`. Buat 7 sections kosong
dengan order numbering 10/20/30/40 (kelipatan 10 supaya gampang
sisip section baru di tengah). `isPublished=false` semua ‚Äî super
admin harus eksplisit publish.

Run: `npx tsx prisma/seedGuides.ts` dari `backend/`.

### Backend service + controller + routes

`backend/src/services/guides.service.ts`:
- `guidesService.list/getById/getBySlug/create/update/unpublish/hardDelete/listRevisions`
- `update()` wrap dalam `prisma.$transaction` ‚Äî kalau `contentMd`
  berubah, snapshot lama insert ke `GuideSectionRevision` SEBELUM
  overwrite. Detect via comparing `existing.contentMd` vs
  `input.contentMd`.
- `guideImagesService.record/getByFilename`

`backend/src/controllers/guides.controller.ts`:
- Multer storage di `uploads/guides/` (relatif working dir backend),
  filter MIME `image/jpeg|png|webp|gif`, limit 2 MB.
- Mapping `audienceForRoles(roles: Set<Role>)` ‚Üí derive audience(s)
  dari role caller untuk endpoint `/my`.
- `serveImage` defense-in-depth: tolak filename yang mengandung
  `/` atau `..` (express normalize URL juga sudah strip `..`,
  tapi double-check); resolve absolute path dan pastikan tetap
  dalam UPLOAD_DIR sebelum sendFile.

`backend/src/routes/guides.routes.ts`:
- `GET /api/guides/image/:filename` ‚Äî public (di atas requireAuth),
  karena `<img src>` tidak boleh butuh auth headers; enumerasi
  filename random tetap aman karena lookup via DB.
- `GET /api/guides/my` & `/audience/:audience` ‚Äî siapa pun yang login.
- `GET/POST/PATCH/DELETE /api/guides[...]` ‚Äî `requireRole("SUPER_ADMIN")`.

Mounted di `/api/guides` via `routes/index.ts`.

### Frontend

Dependencies baru (root `package.json`): `react-markdown`,
`remark-gfm`, `rehype-sanitize`. Tidak pakai `@tailwindcss/typography`
supaya tidak nambah konfigurasi Tailwind v4 yg sudah pakai
`@import "tailwindcss"` style. Custom class `.petunjuk-prose`
di `globals.css` cukup untuk styling heading/list/link/img/table/
code/blockquote.

**Reader** (`src/components/PetunjukReader.tsx`):
- Generic component dengan prop `audience` + `pageTitle` + `breadcrumbLabel`
- Fetch `/api/guides/audience/:audience` ‚Üí state `guides`
- `PageLayout.sidebarItems` = list section titles (vertical sidebar
  yang sudah ada di pattern existing)
- Active section ditampilkan sebagai article dengan
  `<ReactMarkdown remarkPlugins={[remarkGfm]} rehypePlugins={[rehypeSanitize]}>`
- 3 thin wrapper page: `PetunjukMahasiswa.tsx`, `PetunjukDosen.tsx`,
  `PetunjukTendik.tsx` ‚Äî masing-masing ~10 baris.

**Manajemen** (`src/pages/ManajemenPetunjuk.tsx`):
- Default view: tabel section dengan filter audience dropdown +
  tombol "Tambah Section". Kolom: Judul, Audience, Slug, Urutan,
  Status (badge Tayang/Draft), Diperbarui, Aksi (edit/hapus).
- Klik pensil ‚Üí `EditorView` (state `editing` di-set, render
  alternative).
- Klik "Tambah Section" ‚Üí `CreateModal` (audience + judul + slug
  auto-generated dari judul + order). Setelah create, langsung lanjut
  ke EditorView untuk mulai isi konten.
- `EditorView`:
  - Form atas: title/slug/order/isPublished
  - Split-view: textarea kiri (font-mono, 480px height) + preview
    kanan (`react-markdown` dengan rehype-sanitize, scroll independent)
  - Toolbar atas textarea: tombol "Upload Gambar" (input file hidden)
    ‚Üí POST FormData ke `/api/guides/image` ‚Üí response `{ url }` ‚Üí
    `insertAtCursor(\`\\n\\n![\${altText}](\${url})\\n\\n\`)`
  - State `isDirty` deteksi unsaved changes; tombol Simpan disabled
    kalau tidak dirty; show "Belum disimpan" / "Tersimpan HH:mm:ss"
    indicator.
  - Pakai `apiFetch` (cookie credentials, X-HTTP-Method-Override
    untuk PATCH/DELETE supaya lolos Cloudflare WAF block).
  - Image upload pakai `fetch` biasa (bukan apiFetch) karena
    multipart FormData tidak butuh method override.

### Hardening

- Path traversal di image serve (selain express normalize, ada
  manual check filename + abs path containment)
- Markdown sanitization (rehype-sanitize default schema; strip
  `<script>`, event handlers, `javascript:` URLs)
- File size limit 2 MB + MIME whitelist (multer fileFilter)
- RBAC: reader strict per role; admin endpoint `requireRole("SUPER_ADMIN")`
- Slug regex `/^[a-z0-9-]+$/` (zod) ‚Äî tidak boleh slash/dot/space
