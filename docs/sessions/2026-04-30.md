### Fix #1 — Fire-and-forget notif

`backend/src/controllers/equipmentLoans.controller.ts` — 3 spot,
ubah `await notifyEquipmentLoan*(...)` jadi `void
notifyEquipmentLoan*(...)`. Catch handler bawaan dispatcher tetap
log error untuk troubleshooting. Konsisten dengan pola fire-and-forget
yang sudah ada di `loans.controller.ts`.

Komentar 1-line di atas call pertama:
```ts
// Fire-and-forget: SMTP + Fonnte bisa lambat / 502, tidak boleh
// blocking response. Error tetap di-log untuk troubleshooting.
```

Konsekuensi: response 201 langsung balik ~100-300ms (db write + commit
saja), bukan menunggu 2-5 detik untuk SMTP + WA round-trip.

### Fix #2 — Dropdown shadcn (Radix Portal)

`src/pages/PeminjamanAlat.tsx`:

- Import `DropdownMenu`, `DropdownMenuContent`, `DropdownMenuItem`,
  `DropdownMenuTrigger` dari `../components/ui/dropdown-menu`
- Replace block custom (button + conditional `<div>` absolute) dengan
  `<DropdownMenu>` + `<DropdownMenuTrigger asChild>` + `<DropdownMenuContent
  align="end">` + 2 `<DropdownMenuItem>` (Kembalikan, Perpanjang)
- Hapus state `openDropdownIndex` yang tidak dipakai lagi

Mekanisme: shadcn `DropdownMenu` pakai Radix Portal — render ke
`document.body`, **escape** stacking context dan overflow parent.
Plus auto-flip kalau ruang bawah viewport tidak cukup (collision
detection bawaan Radix).

### Fix #3 — Tampilkan kode di Riwayat

`backend/src/services/equipmentLoans.service.ts:18` — tambah `code:

`src/pages/PeminjamanAlat.tsx`:

## 6. Notif kalab setelah dosen approve laptop loan — IMPLEMENTASI (TODO carry-over dari 25 Apr)

Carry-over dari sesi 25 April section 6 — bug dikonfirmasi tapi belum
di-fix. Sesi ini dieksekusi.

### Recap bug

Helper `notifyLoanApprovalToKalab` ada (`notification/index.ts:144-149`)
+ template ada (`templates.ts:155-197`) — tapi **tidak ada satupun
call site di codebase**. Konsekuensi: dosen approve loan → kalab tidak
dapat notif → kalab harus polling sendiri ke modul approval.

### Implementasi

`backend/src/controllers/loans.controller.ts`:

1. **Imports** (line 11-22) — tambah `notifyLoanApprovalToKalab`
2. **Branch `APPROVED_BY_DOSEN`** (line 200-231) — setelah notif ke
   mahasiswa, tambah loop kalab:

```ts
// Laptop global (laboratoryId null) → semua kalab aktif perlu tahu.
// Pola sama dengan loop laboran di branch APPROVED.
const kalabs = await prisma.user.findMany({
  where: {
    isActive: true,
    roles: { some: { role: { name: "KEPALA_LAB" } } },
  },
  select: { displayName: true, email: true, waNumber: true },
});
const waktuPersetujuan = fmtDateTime(new Date());
for (const kalab of kalabs) {
  if (!kalab.email && !kalab.waNumber) continue;
  void notifyLoanApprovalToKalab(
    {
      email: kalab.email ?? undefined,
      phone: kalab.waNumber ?? undefined,
    },
    {
      kalabName: kalab.displayName,
      namaMahasiswa: loan.borrower.displayName,
      nim: loan.borrower.uid,
      disetujuiOleh: actor.displayName,
      waktuPersetujuan,
    },
  );
}
```

### Detail desain

- `prisma.user.findMany` filter `isActive: true` + role `KEPALA_LAB` —
  mirror loop laboran di branch `APPROVED` (line 222-245)
- Skip kalab tanpa email & tanpa WA (`continue`) — hindari dispatcher
  reject silently
- Pakai `void` (fire-and-forget) — konsisten dengan call lain di file
  ini, plus konsisten dengan equipmentLoans yang baru di-fix di section 1
- `waktuPersetujuan = fmtDateTime(new Date())` di-hoist sekali sebelum
  loop, supaya semua kalab dapat timestamp identik (kalau di-call
  per-iterasi, ada risiko ms-level drift)

### Konsekuensi flow notif laptop loan setelah fix

| Event | Notif terkirim ke |
|---|---|
| Mahasiswa apply (PENDING) | Mahasiswa + Dosen pembimbing |
| **Dosen approve (APPROVED_BY_DOSEN)** | **Mahasiswa + semua Kalab aktif** ✅ FIXED |
| Kalab approve (APPROVED) | Mahasiswa + semua Laboran aktif |
| Laboran serah-terima (ACTIVE) | Mahasiswa |
| Pengembalian (RETURNED) | Mahasiswa (+ paragraf surat bebas lab kalau `loan.type === TA`) |

Konsisten dengan flow clearance & room reservation yang notif kalab
langsung setelah laboran/dosen check.
