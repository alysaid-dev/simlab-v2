## Catatan teknis yang perlu diingat

- **Deploy backend** — systemd unit `simlab-v2-be`, perlu `sudo systemctl restart` setelah edit backend (sudah di-memory).
- **Prisma regeneration** — setelah `schema.prisma` diubah: `npx prisma db push --accept-data-loss --skip-generate` lalu `npx prisma generate`.
- **Cloudflare PATCH block dari VPN** — kalau ada 403 HTML saat PATCH dari IP VPN, cek `server: cloudflare` + `cf-ray` dulu sebelum debug RBAC (sudah di-memory).
- **Bucketing logic** — karena backend simpan 1 row per consumable per transaksi, frontend selalu group by `userId + type + menit(createdAt)` supaya 1 transaksi multi-barang tampil 1 baris.
- **Recipient parse regex** di History — `/^Diambil oleh (.+?) \(([^)]+)\)\s*$/`. Dependent pada format otomatis dari `consumables.controller.ts createTransactionBulk`. Kalau laboran isi `notes` manual via endpoint lama per-item, field Penerima di tabel History akan kosong ("-").
