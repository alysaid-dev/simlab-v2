## Hal yang diselesaikan

### 1. Diagnosa cache 301 legacy v1 → /schedule

**Laporan**: buka `statistics.uii.ac.id/simlab` selalu redirect ke
`/simlab/schedule` (yang tidak ada di v2).

**Akar masalah**: browser user masih cache 301 permanent dari era v1
(`/etc/nginx/default.v1.conf:22-24` lama punya `return 301 /schedule`).
Nginx v2 (`/etc/nginx/conf.d/default.conf:28-30`) sudah benar redirect ke
`/simlab/login`. Live curl juga bersih. Verifikasi: dist bundle v2 tidak
punya route `/schedule` (cuma internal React `unstable_scheduleCallback`).

**Solusi**: user clear site data → langsung normal. Tidak perlu patch
nginx. Memory disimpan di `memory/legacy_301_cache.md` — kalau user lain
ngeluh hal serupa, jawaban ready: clear cache, jangan debug nginx/React.

**Risiko tersisa**: user legacy lain kemungkinan masih punya cache sama,
dan Cloudflare edge bisa juga cache. Kalau masalah meluas: ganti 301 →
302 + `Cache-Control: no-store` di `location = /simlab`, plus purge CF.

### 2. Modul "Riwayat Saya" untuk MAHASISWA

**Sebelumnya**: `/history` hanya untuk KEPALA_LAB + SUPER_ADMIN bypass.
Backend service (`history.service.ts`) mengembalikan SEMUA record tanpa
filter userId — yang menjaga privasi cuma guard role di routes.

**Perubahan backend** — scope "Riwayat Saya":
- `backend/src/routes/history.routes.ts` — drop `requireRole(KEPALA_LAB,SUPER_ADMIN)`,
  cukup `requireAuth`. Scoping ditangani service.
- `backend/src/services/history.service.ts` — tambah `HistoryScope`
  interface (`{ isAdmin, isLaboran, userId, userUid }`). Tiap list method
  tambah `WHERE OR [...]` sesuai role:
  - Non-admin: `userId=me`, `lecturerId=me` (loans)
  - LABORAN tambah: `laboranHandoverBy`, `laboranReturnBy`, `signerUidLaboran=uid`, `checkedBy`, consumable `userId=me`
  - Admin: bypass
  - Timeline: cek ownership, 404 kalau bukan punyanya (bukan 403 — biar eksistensi tidak bocor)
- `backend/src/controllers/history.controller.ts` — helper `deriveScope(req)`
  pakai `usersService.getByUid` resolve uid→id, pass ke service tiap endpoint.

**Perubahan frontend**:
- `src/lib/moduleAccess.ts` — awalnya dibuka untuk `["MAHASISWA","DOSEN","STAFF","LABORAN"]`,
  tapi user minta rollout kecil: akhirnya jadi `["MAHASISWA"]` saja.
  KEPALA_LAB dikeluarkan (sudah punya Persetujuan Kepala Lab + Inventaris Lab).
- `src/pages/Dashboard.tsx:117-123` — card title "Riwayat Saya", desc
  "Peminjaman, reservasi, & surat saya".
- `src/pages/History.tsx` — tab "Barang Keluar" sidebar di-hide untuk
  non-LABORAN, title halaman → "Riwayat Saya".

**Catatan**: backend scoping untuk DOSEN/STAFF/LABORAN tetap dibangun
lengkap (defense in depth) meskipun belum dibuka di frontend. Kalau role
itu hit endpoint langsung, tetap dapat scoped result — tidak leak.

### 4. Modul baru "Transaksi Saya" untuk DOSEN + STAFF

**Desain**: dari perspektif peminjam/pengaju (bukan operator). 4 tab:
Peminjaman Laptop, Peminjaman Alat, Habis Pakai, Peminjaman Ruangan.
Menampilkan semua status (aktif + selesai), beda dari "Riwayat Saya"
mahasiswa yang cuma final statuses.

**Konfirmasi mekanisme habis pakai**: laboran input recipient UID di
form, `consumables.controller.ts:127` auto-generate notes
`"Diambil oleh ${displayName} (${uid})"`. Format konsisten untuk semua
role (NIM/NIP/NIK). Scoping frontend → match `notes.contains("(${uid})")`
di Prisma; false positive negligible karena pattern `(uid)` distinctive.

**Backend — 4 endpoint baru di `me.routes.ts` + `me.controller.ts`**:
- `GET /api/me/transactions/laptops` — `Loan WHERE userId=me`, all statuses
- `GET /api/me/transactions/equipment` — `EquipmentLoan WHERE userId=me`, items flattened
- `GET /api/me/transactions/rooms` — `RoomReservation WHERE userId=me`
- `GET /api/me/transactions/consumables` — `ConsumableTransaction WHERE type=OUT AND notes.contains("(${uid})")`

Tidak pakai service baru; ikut pola `me.controller.ts` existing (query
langsung di controller). Punya `listQuery` zod schema sendiri (skip/take).

**Frontend**:
- `src/pages/TransaksiSaya.tsx` — baru. 4-tab sidebar pattern mirip
  History.tsx. Icon Receipt. Badge status warna: aktif=biru, selesai=
  hijau, tolak/batal=merah. Tidak ada timeline modal (v1 lean).
- `src/routes.ts` — route `/transaksi-saya`.
- `src/lib/moduleAccess.ts` — entry `["DOSEN","STAFF"]`.
- `src/pages/Dashboard.tsx` — card "Transaksi Saya" biru-indigo,
  icon receipt.

**Kolom Peminjaman Laptop** (disederhanakan per request user):
Laptop | Tanggal Pinjam | Tanggal Kembali | Status. Kolom Jenis
(TA/Praktikum) & Dosen Pembimbing dihapus karena tidak relevan untuk
DOSEN/STAFF sebagai peminjam.

### 5. Bug fix: race condition antar tab

**Gejala**: `TypeError: Cannot read properties of undefined (reading '0')`
saat switch tab (terutama dari Laptop ke Peminjaman Alat).

**Akar**: initial design pakai single state `rows: AnyRow[]` (polymorphic
union) + type cast `as EquipmentRow[]` di render. Kalau user klik cepat
Laptop → Equipment, fetch laptop yang pertama bisa complete SETELAH tab
sudah switch, timpa state dengan LaptopRow sementara render berada di
equipment branch → akses `r.items[0]` crash karena LaptopRow tidak
punya `items`.

**Fix**: refactor TransaksiSaya.tsx:
- State split per tab: `laptops`, `equipment`, `consumables`, `rooms`
  masing-masing `useState`. Hapus `AnyRow` + type cast.
- `useEffect` pakai `AbortController` + `active` flag. Kalau tab berubah,
  cleanup abort fetch lama; guard `if (!active) return` cegah setter
  dipanggil setelah unmount/tab-change.

Cross-tab contamination: impossible by construction setelah refactor.

## Perubahan sudoers (sambungan sesi kemarin)

Dari sesi part 3 kemarin, setup `/etc/sudoers.d/simlab-restart` NOPASSWD
untuk 5 command (`systemctl restart/start/stop/status simlab-v2-be` +
`journalctl -u simlab-v2-be *`). Hari ini baru ketahuan: rule `status
simlab-v2-be` tidak match kalau dipakai dengan `--no-pager` (sudo match
args secara literal). Workaround saat ini: pakai `systemctl is-active
simlab-v2-be` (tidak perlu sudo) untuk verifikasi service state.
