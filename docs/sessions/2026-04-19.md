## Route protection

- **`ProtectedRoute`** — cek `useAuth().loading` & `user`. Loading → spinner full-screen. `!user` → `<Navigate to="/login">`.
- **`PublicOnlyRoute`** — kebalikannya; user yang sudah login & buka `/` atau `/login` → redirect ke `/dashboard`.
- `routes.ts`: semua route modul di-wrap `guarded(...)`; `/` + `/login` pakai `publicOnly(...)`; `/verify/:hash` tetap publik.

## Opsi A — Lock aset pada PENDING

- `loans.service.create`:
  - Pre-check `Asset.status === AVAILABLE` → lempar `409` kalau sudah di-lock.
  - Lock ke `BORROWED` bareng create loan (transaksi Prisma).
- `updateStatus` sudah handle release: `REJECTED/CANCELLED/RETURNED` → `AVAILABLE` (hanya jika `BORROWED`, hindari override DAMAGED/MAINTENANCE).
- **Backfill SQL** saat eksekusi — LT-003 & LT-007 ter-update ke BORROWED.
- Enum tidak ditambah `PENDING`/`RESERVED` — pakai BORROWED sebagai state "ter-lock". Filter `?status=AVAILABLE` tetap benar.

## Super Admin — Monitor Transaksi

- `GET /api/auth/me` kini return `roles: Role[]` (dari `deriveRoles(shib)`).

## Surat Bebas Lab — End-to-End

### Schema (`prisma db push`)

`ClearanceLetter` tambah 8 field:
- `tanggalSidang` (DateTime?)
- `rejectionReason` (Text?)
- `nomorSurat` (String? @unique) — format `{urut3}/Lab.Stat/FMIPA-UII/{bulan_romawi}/{tahun}`
- `signerUidLaboran`, `signerUidKepalaLab`
- `hashLaboran` (@unique), `hashKepalaLab` (@unique)
- `signedAtLaboran`, `signedAtKepalaLab`
- `pdfUrl`

Enum `ClearanceStatus` tidak diubah — pakai existing:
`PENDING_LABORAN → PENDING_KEPALA_LAB → APPROVED | REJECTED`.

### Backend

- **`clearancePdf.service.ts`** (baru):
  - `generateHash(id, uid, ts)` — SHA-256(`${id}${uid}${isoTs}${QR_SECRET_KEY}`)
  - `nomorSurat(urut, date)` — format standar dengan bulan Romawi
  - `renderClearancePdf(...)` — pdfkit A4 1-halaman, logo dari `src/assets/logo-atas-surat.png` (fit 200×90pt, centered top), judul + nomor + data mahasiswa + paragraf + 2 QR + nama signer + NB
- **`clearances.service.ts`** (rewrite):
  - `approveByLaboran(id, uid)` → `PENDING_KEPALA_LAB` + signedAtLaboran + hashLaboran
  - `approveByKepalaLab(id, uid, approverId)` → `APPROVED` + nomor + PDF + pdfUrl
  - `reject(id, reason)`, `ensurePdf(id)` (regenerate kalau hilang)
- **`clearances.controller.ts`**:
  - `create` → notif mahasiswa + SEMUA laboran aktif
  - `updateStatus`:
    - `PENDING_KEPALA_LAB` (laboran approve) → notif mahasiswa + semua kalab
    - `APPROVED` (kalab approve) → generate PDF → email mahasiswa **dengan PDF lampiran** + WA
    - `REJECTED` → template rejected dengan `rejectionReason` + deteksi tahap (Laboran/KepalaLab)
  - `download` → `GET /api/clearances/:id/download` (guard: owner atau LABORAN+)
- **Notif transport**: `sendEmail` dukung `attachments` (filename + path + contentType) via nodemailer.
- **Templates baru**: `clearanceCreatedToLaboran`, `clearanceCheckedToKepalaLab`, `clearanceRejectedToMahasiswa`.
- **Verify**: prioritas lookup `ClearanceLetter.hashLaboran/hashKepalaLab` dulu; fallback ke `DigitalSignature`. Role label "Laboran" kalau hashLaboran, "Kepala Laboratorium" kalau hashKepalaLab.

### Frontend

- `SuratBebasLab.tsx`: tombol Unduh pakai endpoint `/download`.
- `PersetujuanLaboran.tsx`: setujui kirim `PENDING_KEPALA_LAB` (bukan APPROVED). Tolak wajib alasan.
- `PersetujuanKepalaLab.tsx`: setujui kirim `APPROVED` (bukan PENDING_LABORAN lagi). Tolak wajib alasan + pakai `rejectionReason` (bukan notes).

### Env / config / libs

- `.env`: `QR_SECRET_KEY` (32-byte hex random), `STORAGE_ROOT`, `PUBLIC_BASE_URL`.
- `config/env.ts`: field `qrSecretKey`, `storageRoot`, `publicBaseUrl`.
- Libs: **pdfkit + qrcode** (pure-JS, zero system deps — puppeteer ditolak karena butuh Chromium + libnss3 dkk).
- Folder: `backend/storage/clearances/` (PDF dummy di sini).
