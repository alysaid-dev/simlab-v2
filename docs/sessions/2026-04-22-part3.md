## 5) RBAC audit ronde 1 — 8 temuan 🔍

Inventarisasi middleware (`auth.ts`), scan 17 route file + 5 controller, cross-check dengan `moduleAccess.ts` frontend.

### 🔴 Kritis

**#1. `requireRoleAtLeast("LABORAN")` diam-diam lolos untuk DOSEN**

Root cause: `ROLE_RANK` di `auth.ts:46-53` menempatkan `DOSEN=3` di atas `LABORAN=2`. `hasRoleAtLeast(user, "LABORAN")` check `rank >= 2`, jadi DOSEN (rank 3) lolos.

Endpoint yang kena: `POST/PATCH/DELETE /api/users`, `PUT /api/users/:id/roles`, `GET /api/loans/fines`, `PATCH /api/loans/:id/fine`, `POST/PATCH/DELETE /api/equipment`.

**Jalur escalation nyata:** DOSEN login SSO → `curl -X PUT .../api/users/<own-id>/roles -d '{"roles":["SUPER_ADMIN"]}'` → `requireAuth` lolos, `requireRoleAtLeast("LABORAN")` lolos (DOSEN rank 3), controller `replaceRoles` tidak punya guard tambahan → DOSEN jadi SUPER_ADMIN setelah session cache TTL expire.

Dikonfirmasi dengan baca `usersController.replaceRoles` — memang passthrough ke service tanpa guard tambahan.

**#2. `/api/test/**` tanpa auth di production**

`routes/index.ts:42-43` mount `testRoutes` tanpa guard. `test.routes.ts` **tidak punya** `requireAuth`. `POST /api/test/send-notification` menerima `{template, email, phone}` arbitrary dan kirim email/WhatsApp via sistem SIMLAB (17 template termasuk peniru persetujuan dosen/kalab) — phishing/spam gateway untuk siapapun yang bisa hit API.

Comment "REMOVE BEFORE PRODUCTION" sudah ada sejak awal tapi tidak pernah dicabut.

### 🟡 Tinggi (privacy / data leak)

**#3. List endpoint tidak auto-scoping ke requester**

`getById` sudah punya `isOwner || LABORAN+` check, tapi `list` hanya passthrough query. `GET /api/loans`, `/clearances`, `/reservations`, `/equipment-loans` tanpa `?userId=X` kembalikan SEMUA record dari semua mahasiswa. Frontend memang convention-nya pass `?userId=<me>`, tapi siapapun yang bisa curl API langsung bisa scrape seluruh riwayat lab.

### 🟠 Sedang

**#4. Asset writes mengizinkan KEPALA_LAB** — memory `lab_domain_rules.md` bilang kalab read-only di inventaris, tapi `assets.routes.ts` membuka POST/PATCH untuk `LABORAN, KEPALA_LAB, SUPER_ADMIN`.

### 🟢 Observasi

**#5. Dua style guard di-mix** — `requireRole` (whitelist eksak) vs `requireRoleAtLeast` (threshold hierarchy). Kalau semua whitelist, bug #1 tidak akan terjadi.

### ✅ Yang sudah benar

- `shibbolethAttach` global, `requireAuth` konsisten.
- `getById` untuk users/loans/clearances/reservations/equipmentLoans sudah punya owner-or-LABORAN+ check.
- `/api/me/active-items` self-scoped ke requester.
- `laboratories.PATCH`, `app-settings.PATCH`, `consumables.DELETE` → `requireRole("SUPER_ADMIN")` eksak.
- `deriveRoles()` pakai exact match (bukan substring) — tidak bisa dibypass via group name tipuan.
- SUPER_ADMIN bootstrap via env — clean.

## 7) RBAC audit ronde 2 — 7 temuan BARU 🔍

User minta audit dimensi lain di luar RBAC. Sweep: middleware stack, file upload/download, info disclosure, state machine, secrets, QR public endpoint, rate-limit.

### 🔴 Kritis (baru)

**#6. `PATCH /api/loans/:id/status` tanpa role guard + tanpa state machine**

`loans.routes.ts:19` — `router.patch("/:id/status", loansController.updateStatus)` tanpa `requireRole`. Service `loans.service.ts:97` (`updateStatus`) langsung tulis status tanpa validasi transisi.

**Efek:** mahasiswa manapun bisa PATCH loan apapun (tidak hanya miliknya) ke status apapun. Skenario: buat loan → langsung set ACTIVE (skip approval chain); atau set ke RETURNED untuk clear obligations sebelum apply surat bebas lab.

`PATCH /api/loans/:id` (line 25) juga tanpa guard — `endDate` dan `notes` mutable oleh siapapun. Mahasiswa bisa perpanjang loan siapapun tanpa persetujuan.

Domain lain (clearances, reservations, equipmentLoans) **sudah benar** — semuanya guard `updateStatus` dengan `requireRole`. Hanya `loans` terlewat.

**#7. `NODE_ENV=development` di server production**

`backend/.env`: `NODE_ENV=development`. `isDev = env.nodeEnv !== "production"` jadi `true`. `errorHandler.ts:108-109` bocor stack trace + raw error message di setiap 500 error. Morgan juga verbose (`"dev"` format instead of `"combined"`).

Fix 30 detik: ganti satu env var, restart.

### 🟡 Tinggi (baru)

**#8. Tidak ada rate limiter sama sekali**

`grep express-rate-limit` = nol. `POST /loans`, `POST /clearances`, `POST /reservations` — semua trigger notifikasi email + WA. Satu mahasiswa iseng bisa flood inbox staff + drain kuota Fonnte.

**#9. Prisma errors bocor `err.message` mentah**

`errorHandler.ts:74-77` — kalau Prisma error bukan P2002/P2025, dump `err.message` apa adanya (mengandung SQL-ish + nama kolom + value). Bahkan dengan `NODE_ENV=production`, leak ini tetap terjadi.

**#10. QR hash pakai SHA-256 telanjang, bukan HMAC**

`clearancePdf.service.ts:27-30`:
```ts
crypto.createHash("sha256")
  .update(`${clearanceId}${signerUid}${timestamp.toISOString()}${env.qrSecretKey}`)
```
Idiom salah — harusnya `createHmac("sha256", secret)`. SHA-256-append-secret rentan length-extension secara teoretis. Attack practical di kasus ini sulit (secret di posisi akhir), tapi idiom benar murah untuk diterapkan.

### 🟢 Sedang / observasi

**#11. Multer fileFilter trust client-set mimetype + extension** — `reservations.controller.ts:34`. Saat ini file tidak di-serve publik (nginx tidak expose `backend/uploads/`), risk rendah. Kalau nanti bikin endpoint download, wajib validate magic bytes.

**#12. CSRF latent via `X-HTTP-Method-Override`** — Cloudflare WAF workaround (POST + header override = PATCH). Preflight CORS biasanya mencegah cross-origin abuse, tapi kalau ada UI embed di domain sama perlu hati-hati.

### ✅ Yang sudah benar (verified)

- `.env` gitignored (hanya `.env.example` committed).
- `QR_SECRET_KEY` di-set di prod (bukan dev default).
- `SHIBBOLETH_DEV_MOCK=0` di prod.
- Helmet aktif (CSP sengaja off karena SSO).
- CORS tight (origin eksplisit dari env).
- Uploaded files tidak di-serve public.
- `res.download(path.resolve(pdfPath), ...)` — path dari DB, aman dari traversal.
- `actor = usersService.getByUid(req.user!.uid)` — actorId dari session, tidak trust body.
- `suratPermohonanPath` body field dioverride oleh `file.path` multer.
