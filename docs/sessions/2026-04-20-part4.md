### Pendekatan yang dipilih

- `ClearanceLetter` & `RoomReservation` sudah punya per-signer fields lengkap — tidak perlu schema change.

### Schema change — `Loan`
Field baru (semua nullable, loan historis = NULL):
```
dosen_decision, dosen_decision_at
kalab_decision, kalab_decision_at, kalab_decision_by
laboran_handover_at, laboran_handover_by
laboran_return_by
rejection_reason
return_condition (AssetCondition), return_note
```
+ 3 relasi User baru: `LoanKalabApprover`, `LoanLaboranHandover`, `LoanLaboranReturn`.
Migrasi via `npx prisma db push --skip-generate`.

### Auto-sync kondisi aset
- Saat `updateStatus` dipanggil dengan `status = RETURNED` + `returnCondition`, `Asset.condition` ikut di-update supaya peminjaman berikutnya lihat kondisi terkini. Admin tetap bisa override manual di modul Aset.

## 3. Fix modul Aset — eye-modal pakai mock data

### Bug
- Asset ID 5100015 statusnya BORROWED, borrower **ALFANDI HANDY FABIAN** (NIM 22611139), loan ACTIVE.
- Tapi modal "mata" di Aset tidak menampilkan "Peminjam Saat Ini" maupun "Riwayat Peminjaman".

### Root cause
- `src/pages/Aset.tsx` masih pakai `mockCurrentBorrower` & `mockBorrowingHistory` hardcoded untuk ID `"AST-003"`, `"AST-007"`. Tidak pernah di-wire ke backend.

### Fix
- Hapus mock data, tambah `useEffect` fetch `GET /api/loans?assetId={dbId}&take=200` saat `viewingAssetDetail` berubah.
- Split hasil: `ACTIVE`/`OVERDUE` → "Peminjam Saat Ini"; `RETURNED`/`REJECTED`/`CANCELLED` → "Riwayat Peminjaman". `PENDING`/`APPROVED*` diabaikan (belum serah-terima).
- Backend: tambah `assetId` di `loans.service.list` + zod `loans.controller` listQuery.
